Github Pages

先到熟悉到不能在熟悉的github網站新增一個新的repository
將此專案名取為 username.github.io。(username為你帳號的名字)

事前準備

安裝所需：

在這以下動作以前，請先下載：

  1. Node.js：https://nodejs.org/en/
  2. Git：https://git-scm.com/
  3. Notepad++：https://notepad-plus-plus.org/downloads/

事前準備

工具們都下載好後，緊接著執行：

npm install -g hexo-cli //安裝Hexo套件。
mkdir name //建立一個資料夾，叫name。
hexo init name/ //初始化hexo，剛剛有創建一個叫name的資料夾。

事前準備

cd name/
npm install //安裝Hexo
npm install hexo-deployer-git –save //安裝 git 部署套件
成果如下：

設定部落格相關資訊

打開剛建立好的Hexo資料夾，找到 _config.yml 的檔案，利用剛才下載的Nodepad++，修改6-12行的資料。
修改內容如下：

  1. title: 我的部落格 //輸入部落格標題
  2. subtitle: 一個抒發本人心情的地方 //輸入副標題
  3. description: //輸入網站描述

設定部落格相關資訊

  1. keywords: //輸入網站關鍵字(以逗號隔開)，方便 SEO
  2. author: 王小明 //輸入姓名或暱稱
  3. language: zh-TW //輸入您所使用的語言
  4. timezone: //留空以使用系統時間

設定部落格相關資訊

接著再到16行url的那行，改成自己Github Page的網址，通常會是http://name.github.io/

結果如下：

這樣就完成部落格的初步個人化設定囉~~

設定部署至 GitHub 的資訊

同樣在 _config.yml 的檔案中，找到 deploy (蠻下面的XD

type : git
repo : 下圖網址名字的地方改成自己repository的名稱
branch : master

改完後長這樣~~

部署至 GitHub

接著回到git bash中，打上這三行程式

hexo cl //清除之前建立的靜態檔案，也可輸入 hexo clean
hexo g //建立靜態檔案，也可輸入 hexo generate
hexo d //部署至 Github Pages，也可以輸入 hexo deploy

成果如下圖：

一個基本部落格就完成了~

在瀏覽器上打上http://username.github.io/
